# 键盘快捷键说明

## ⌨️ 快捷键映射

### 评分操作

```
┌─────────────────────────────────────┐
│  A 键  →  选择视频A                 │
│  W 键  →  选择视频B                 │
│  D 键  →  两者相当                  │
└─────────────────────────────────────┘
```

### 导航操作

```
┌─────────────────────────────────────┐
│  Q 键  →  上一题（查看/修改历史）   │
│  E 键  →  下一题（返回更新的任务）  │
└─────────────────────────────────────┘
```

---

## 💡 设计理念

### 键盘布局

快捷键设计基于键盘的 **QWEASD** 区域，这是游戏中常用的操作区域：

```
键盘布局：
┌───┬───┬───┬───┐
│ Q │ W │ E │ R │  ← 导航层
├───┼───┼───┼───┤
│ A │ S │ D │ F │  ← 操作层
└───┴───┴───┴───┘

使用的按键：
┌───┬───┬───┐
│ Q │ W │ E │  Q=上一题, E=下一题
├───┼───┼───┤
│ A │   │ D │  A=视频A, W=视频B, D=相当
└───┴───┴───┘
```

### 优势

1. **单手操作**：所有按键集中在键盘左侧
2. **符合直觉**：
   - A（左边）对应视频A（左边）
   - D（右边）对应相当（右边选项）
   - W（中间偏上）对应视频B（中间选项）
3. **游戏化体验**：WASD/QE 是游戏玩家熟悉的操作方式
4. **高效流畅**：
   - 左手操作快捷键
   - 右手操作鼠标/触控板（如需要）
   - 视线无需离开屏幕

## 🎮 使用技巧

### 基本评测流程

```
1. 观看三个视频
   ↓
2. 做出判断
   ↓
3. 按快捷键提交
   - 按 A → 选择视频A
   - 按 W → 选择视频B
   - 按 D → 两者相当
   ↓
4. 自动跳转到下一题
```

### 历史回顾流程

```
1. 完成一些评测后
   ↓
2. 按 Q → 返回上一题
   ↓
3. 查看之前的评测结果
   ↓
4. 继续按 Q 可以持续往前翻
   ↓
5. 按 E 返回到更新的任务
   ↓
6. 如需修改：重新选择（A/W/D）并提交
```

### 快速评测技巧

**场景1：快速连续评测**
```
观看 → 按A → 观看 → 按W → 观看 → 按D → ...
```

**场景2：仔细对比后评测**
```
观看参考视频 → 观看视频A → 观看视频B → 
再次对比 → 按快捷键提交
```

**场景3：修改历史评测**
```
按Q返回 → 再按Q继续往前 → 找到目标 → 
重新选择(A/W/D) → 自动返回当前任务
```

## 📊 快捷键记忆法

### 方法1：位置记忆

```
键盘上的位置对应选择位置：
┌─────────────────────────────┐
│  [A]           →  左边视频   │
│     [W]        →  中间视频   │
│        [D]     →  右边选项   │
└─────────────────────────────┘
```

### 方法2：字母联想

- **A** = Answer A (答案A) / First (第一个)
- **W** = Win (获胜) / Second (第二个)
- **D** = Draw (平局) / Same (相同)
- **Q** = back (Quit = 退出当前, 返回上一题)
- **E** = forward (Enter = 进入, 下一题)

### 方法3：游戏玩家记忆

如果你玩过WASD游戏：
- **Q/E** = 传统的切换/旋转键（在这里用于导航）
- **A/D** = 左右移动键（对应左边视频A和右边选项D）
- **W** = 前进键（选择视频B）

## 🔧 技术实现

### 工作原理

1. **隐藏输入框**：创建一个1x1像素的透明输入框
2. **自动聚焦**：每100ms检查并确保输入框获得焦点
3. **按键捕获**：监听 `keydown` 事件
4. **按钮查找**：通过文本内容查找对应的按钮
5. **模拟点击**：触发按钮的点击事件

### 控制台日志

打开浏览器开发者工具（F12），可以看到：

```javascript

Clicking button: 选择 视频A
```

## ⚠️ 注意事项

### 快捷键不响应的情况

1. **焦点在其他输入框**：如果点击了页面上的其他输入框，快捷键可能暂时失效
   - 解决：点击视频区域或页面空白处
   
2. **按钮被禁用**：导航按钮在边界时会被禁用
   - Q键：已经是第一题时无效
   - E键：已经在当前任务时无效

3. **浏览器快捷键冲突**：某些浏览器可能有默认快捷键
   - 通常不会冲突，但如果遇到可以尝试其他浏览器

### 最佳实践

1. **保持专注**：视线集中在视频和评价按钮上
2. **养成习惯**：多用几次就能形成肌肉记忆
3. **合理导航**：
   - 不确定时使用Q返回查看
   - 发现错误立即修改
   - 评测完成后用Q回顾抽查

## 💬 常见问题

**Q: 为什么不是S而是W？**
A: S键在键盘上的位置会让手指有点尴尬，W键更符合游戏操作习惯，且位置更顺手。

**Q: 可以自定义快捷键吗？**
A: 当前版本暂不支持，如有需求可以联系开发者。

**Q: 快捷键在移动设备上能用吗？**
A: 移动设备通常没有物理键盘，建议使用触摸操作。

**Q: 能否支持数字键盘？**
A: 主键盘区的数字键和字母键都支持，小键盘区的数字键可能不支持。

**Q: 大写锁定（Caps Lock）会影响吗？**
A: 不会影响，程序已处理大小写兼容。
